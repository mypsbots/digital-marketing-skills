# End-to-end examples

These worked examples show how the skills, prompts and workflows fit together for realistic
scenarios. They use **illustrative, synthetic numbers** — not benchmarks or guarantees — to
demonstrate method and structure. Any spend, publishing or customer contact in a real engagement
must pass the platform's human-approval gates.

| Example | Scenario | Prompts / workflows used |
| --- | --- | --- |
| [SaaS growth plan](./saas-growth-plan.md) | PLG SaaS wants to grow activation and expansion | `saas_growth_plan`, `marketing-strategy-development` |
| [E-commerce Google Ads plan](./ecommerce-google-ads.md) | D2C store plans a margin-aware paid-search launch | `plan_google_ads`, `google-ads-launch` |

Each example follows the same shape: **context → objective → method (skills applied) →
illustrative output → measurement → safety/approvals**.
